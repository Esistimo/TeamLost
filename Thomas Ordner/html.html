Welche Funktion hat ein Browser?

--> spezielle Computerprogramme zur Darstellung von Webseiten im World Wide Web oder allgemein von Dokumenten und Daten

Welche Browser gibt es und wie unterscheiden sie sich?

Google Chrome, Mozilla Firefox, Apple Safari, Microsoft Edge, Opera.
Die Unterschiede zwischen den Browsern liegen in ihren Funktionen, der Geschwindigkeit, der Sicherheit, der Anpassbarkeit und der Benutzerfreundlichkeit.
Einige Browser sind auch besser für bestimmte Aufgaben geeignet als andere.
Zum Beispiel ist Firefox besser für Entwickler geeignet, während Safari besser für Apple-Benutzer geeignet ist.

Wie geht ein Browser mit einer HTML Seite um?

1. Der Browser lädt die HTML-Datei vom Webserver herunter, um den Inhalt (HTML-Tags, Inhalt, Struktur und Layout) der Seite zu definieren und zu analysieren.

2. Danach wandelt der Browser ihn in eine visuelle Darstellung um.

3. Dann wird ein DOM (Document Objekt Model) erstellt, das die Struktur der Seite in ein Baumdiagramm darstellt.
Es ermöglicht dem Browser, auf die verschiedenen Elemente und Attribute der Seite zuzugreifen, um sie zu ändern.

4. Als Nächstes ermöglicht der Browser dem Benutzer, auf verschiedene Elemente der Seite zuzugreifen, indem er Links anklickt oder Formulare ausfüllt.
Der Browser verarbeitet diese dann vom Benutzer und sendet sie an den Webserver zurück, um die entsprechenden Aktionen durchzuführen.

Was passiert beim Parsen?

Beim Parsen von HTML wird der vom Browser heruntergeladene HTML-Code analysiert und in ein sogenanntes "Document Object Model" (DOM) umgewandelt, das die Struktur und das Layout der Seite darstellt. Der Parsing-Prozess besteht aus mehreren Schritten:

1. Lexikalische Analyse: Der Parser liest den HTML-Code Zeichen für Zeichen und erzeugt dabei eine Folge von Tokens, die den verschiedenen HTML-Elementen und -Attributen entsprechen.

2. Syntaxanalyse: Der Parser analysiert die Tokenfolge und erzeugt daraus eine Syntaxstruktur, die die Hierarchie der HTML-Elemente und ihre Beziehungen untereinander darstellt.

3. Syntaxanalyse: Der Parser analysiert die Tokenfolge und erzeugt daraus eine Syntaxstruktur, die die Hierarchie der HTML-Elemente und ihre Beziehungen untereinander darstellt.

4. Rendering: Nachdem der Parser das DOM erstellt hat, verwendet der Browser die Informationen im DOM, um die Seite zu rendern und anzuzeigen. Dabei werden auch CSS-Stylesheets und JavaScript-Dateien berücksichtigt, die in der HTML-Datei referenziert werden.

Während des Parsing-Prozesses prüft der Parser auch auf syntaktische Fehler im HTML-Code. Wenn der Parser auf einen Fehler stößt, wird er versuchen, den Fehler zu korrigieren, um den Code so gut wie möglich zu interpretieren. Wenn jedoch ein schwerwiegender Fehler auftritt, kann es sein, dass der Parser die Seite nicht richtig rendern kann oder sogar abstürzt.
